# 07. Preferences Form

You are given a settings form with a few gaps.

Implement:

1. Client-side validation.
2. Dirty-state tracking.
3. Async save with loading and error handling.
4. Reset back to the last saved values.

Potential issues:

- The starter mixes controlled and uncontrolled inputs.
- The current submit flow always "succeeds" visually.
- Be ready to explain how you would type reusable form state helpers.
