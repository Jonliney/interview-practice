# 19. Interview Scheduler

You are given an interview scheduler with a list of available slots and saved bookings.

Implement:

1. Displaying slot times in the viewer's local timezone.
2. Creating a booking without allowing overlapping bookings for the same interviewer.
3. Editing and canceling an existing booking.
4. Clear feedback when a requested slot is invalid.

Notes:

- All stored times are ISO strings in UTC.
- Be ready to explain how you would test timezone-sensitive logic.
